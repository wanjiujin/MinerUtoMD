# MinerUtoMD 部署/安装说明

本文档用于在新电脑上部署 MinerUtoMD，或把当前方案交给别人复用。

## 1. 环境要求

- Windows 10/11
- Python 3.10 或 3.11（建议使用 Conda/Miniforge 独立环境）
- MinerU（用于 PDF 解析）
- Pandoc（用于 Markdown 转 Word/HTML/EPUB/PDF）
- wkhtmltopdf、xelatex 或 weasyprint 之一（需要导出 PDF 时使用）
- 可选：NVIDIA GPU + CUDA 版 PyTorch（大批量 PDF、公式识别会更快）

## 2. 获取项目

```powershell
git clone https://github.com/wanjiujin/MinerUtoMD.git
cd MinerUtoMD
```

如果是直接拷贝压缩包，也请先进入项目根目录。

## 3. 创建 Python 环境

示例使用 Conda/Miniforge：

```powershell
conda create -n minerutocvt python=3.10 -y
conda activate minerutocvt
pip install -r requirements.txt
```

然后按 MinerU 官方说明安装 MinerU。安装完成后确认下面命令至少有一个可用：

```powershell
mineru --help
magic-pdf --help
```

## 4. 安装 Pandoc 和 PDF 引擎

安装 Pandoc 后确认：

```powershell
pandoc --version
```

如需导出 PDF，建议安装 wkhtmltopdf，并确认：

```powershell
wkhtmltopdf --version
```

如果 Pandoc 或 wkhtmltopdf 没有加入 PATH，也可以在初始化时让脚本探测并写入绝对路径。

## 5. 初始化本机配置

项目内的 `MinerUtoMD/config.yaml` 是模板配置，不再硬编码某一台电脑的 D 盘路径。

先执行环境检查：

```powershell
python MinerUtoMD/init_environment.py
```

确认检测结果后写入本机配置：

```powershell
python MinerUtoMD/init_environment.py --write-config
```

脚本会：

- 探测 Python/MinerU CLI/Pandoc/wkhtmltopdf
- 探测 HuggingFace/MinerU 本地模型缓存
- 自动备份旧的 `config.yaml`
- 把当前电脑可用路径写入 `config.yaml`

如果不想覆盖模板，也可以写到单独文件：

```powershell
python MinerUtoMD/init_environment.py --write-config --output-config MinerUtoMD/config.local.yaml
python MinerUtoMD/main.py -c MinerUtoMD/config.local.yaml doctor
```

## 6. 检查环境

```powershell
python MinerUtoMD/main.py doctor
```

如果看到 Pandoc、MinerU、模型缓存等检查项通过，就可以开始转换。

## 7. 启动 GUI

源码方式启动：

```powershell
python MinerUtoMD/gui_simple.py
```

如果使用 exe，把 `MinerUtoMD.exe` 放在同一目录下，并确保旁边有适合当前电脑的 `config.yaml`。

## 8. 命令行转换示例

PDF 工作流：

```powershell
python MinerUtoMD/main.py convert "input.pdf" -o "output" -f docx -f pdf
```

Word 工作流：

```powershell
python MinerUtoMD/main.py word "input.docx" -o "output" -f docx -f pdf
```

批量转换可在 GUI 中选择文件夹，也可使用命令行批处理功能。

## 9. 本地模型/离线部署

如果需要离线运行：

1. 先在有网络的电脑下载好 MinerU 模型。
2. 把模型缓存目录复制到目标电脑。
3. 准备 `mineru_local_models.json`。
4. 在 `config.yaml` 中设置：

```yaml
mineru:
  model_source: "local"
  mineru_tools_config_json: "你的本地 mineru_local_models.json 路径"
  hf_home: "你的 HF 缓存目录"
  hf_hub_cache: "你的 HF hub 缓存目录"
```

也可以让 `init_environment.py --write-config` 自动写入检测到的本地模型配置。

## 10. 打包 exe

在项目根目录执行：

```powershell
python -m PyInstaller MinerUtoMD.spec
```

打包完成后，exe 通常位于：

```text
MinerUtoMD/dist/MinerUtoMD.exe
```

注意：exe 不包含 MinerU 模型、Pandoc、wkhtmltopdf 等外部环境。换电脑使用时，仍需在目标电脑安装或配置这些依赖。

## 11. 常见问题

### Pandoc 路径不存在或权限拒绝

运行：

```powershell
python MinerUtoMD/init_environment.py
python MinerUtoMD/init_environment.py --write-config
```

如果仍失败，手动确认 `pandoc.exe` 是否存在、是否可执行，或把 Pandoc 加入 PATH。

### GUI 选择 DOCX + PDF 后没有 PDF

通常是 Pandoc 可用但 PDF 引擎不可用。安装 wkhtmltopdf 后重新运行初始化脚本。

### 英文单词粘连、排版不规整

这是 PDF 版式解析常见问题。可尝试：

- 扫描件使用 OCR 模式
- 原生 PDF 使用 txt 或 auto 模式
- 开启表格优化
- 对复杂表格保留 HTML 表格
- 对极复杂版式先输出 Markdown，再人工检查后转 Word/PDF
